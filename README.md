# Bright-TV-analysis
BrightTV's CEO set an objective to grow the company's subscription base for the financial year and approached the analytics team to provide insights that would help the CVM (Customer Value Management) team meet this goal. This case study analyses BrightTV's user profiles and viewership transactions to understand who is watching, when they watch, what factors influence consumption, and where the biggest opportunities lie to grow the subscriber base. The dataset contained two tables — User_Profiles (UserID, Gender, Age, Race, Province) and Viewership (UserID, Channel, RecordDate, Duration) — where each row in Viewership represents a single streaming session, and all dates and times were supplied in UTC and had to be converted to South Africa Standard Time (Africa/Johannesburg). The analysis began with cleaning nulls and blank values in the Province, Race, and Gender columns, followed by joining the two tables on UserID and engineering new features such as month name, day name, viewing-hour buckets (Morning, Afternoon, Evening) and age groups (Minor, Youth, Adult, Senior). Aggregations were then performed to count views, count unique users, and sum total consumption duration across dimensions like Province, Channel, Gender, Age Group, Race, and Viewing Hour Bucket — producing a single analytics-ready dataset used to answer the CEO's questions and recommend content and growth initiatives for the CVM team. To deliver the project end-to-end, I used Miro for diagram planning and mapping out the solution architecture, Canva for building the project timeline and duration plan, Databricks (Spark SQL) for all data cleaning, transformation, and analytical queries, Excel for creating the visuals and dashboard charts, and PowerPoint for compiling and delivering the final 20-minute stakeholder presentation.
